## Work Breakdown Structure (WBS)

### 1. Planning

#### 1.1 Project Setup

* Project idea and scope finalized

---

### 2. Design

#### 2.1 UI/UX Design

* Figma layout and UI design completed

---

### 3. Development

#### 3.1 Core Features

* Core app views built (Dashboard, Add Session, All Sessions, Stats)
* localStorage and live updates implemented

#### 3.2 Additional Features

* Filtering functionality implemented
* Delete feature added
* Statistics/chart feature completed

---

### 4. Testing & Final Delivery

#### 4.1 Testing

* Responsive design adjustments completed
* Final testing performed

#### 4.2 Final Submission

* GitHub repository and README prepared
* Video presentation recorded and submitted

---

## Explanation

I organized the project into four main phases: Planning, Design, Development, and Testing & Final Delivery. This helped break the project into smaller and more manageable parts instead of working on everything at once.

Each phase groups related tasks together. For example, all design work is placed under the Design phase, while building features is grouped under Development. I separated testing and final submission tasks to make sure the project is complete and ready before delivery. This structure made it easier to understand the flow of the project and track progress step by step.

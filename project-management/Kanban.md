## Kanban Board

### Board Columns

For this project, I used four columns:

| Column      | Purpose                         |
| ----------- | ------------------------------- |
| Backlog     | Tasks planned for the project   |
| To Do       | Tasks ready to start            |
| In Progress | Tasks currently being worked on |
| Done        | Completed tasks                 |

### Process Flow

Tasks started in the Backlog column while I was planning the project. When I was ready to work on a task, it moved to To Do, then In Progress while I was building or testing it. After each task was completed and checked, it moved to Done.

---

### Kanban Summary

The Kanban board helped me see the project workload in a simple way. It made it easier to track what still needed to be done, what was being worked on, and what was already finished. Since this was a small solo project, most tasks moved one at a time, but the board still helped me stay organized and avoid missing important parts of the project.
